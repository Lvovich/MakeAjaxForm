v. 0.2.0
- Для создания аякс-формы надо создавать объект вместо простого вызова функции (new MakeAjaxForm). Это на случай, когда
  на странице необходимо несколько форм.
- Все параметры перенесены в один объект (объединены opts и funcs).
- Метод отправки запросов теперь только post, кодировка - только multipart/form-data (не надо в атрибутах контейнера
  указывать эти данные)
- onExchangeError предоставляет в параметре ({string} 'no_valid_data') при отсутствии ошибок валидации и данных.
- Ошибки валидации разделены на 2 типа: 'required_wo_value', 'required_wo_check'. Расширены условия валидации для input
  type="checkbox || radio". Теперь они могут быть required.

v.0
- Решил зафигачить js-плагин для удобного обмена данными ajax-ом. Заодно чтобы формы не было, как таковой, на странице -
  защита от спам-ботов. Идея в том, что плагину передается класс или id контейнера input'ов и класс или id отправляющего
  элемента. По клику на отправляющий элемент, плагин собирает данные всех input'ов в контейнере и отправляет ajax-ом на
  адрес (тоже указывается в параметрах).
